# Cyber Security Internship – Task 14

## Task Title
Linux Server Hardening & Secure Configuration

## Objective
To understand Linux server hardening techniques and apply secure configuration practices to reduce system attack surface.

## Tools Used
- Linux OS (Ubuntu / Kali – conceptual)
- Lynis (Conceptual)
- CIS Benchmarks (Conceptual)

## Hardening Steps Covered
- User and permission management
- Least privilege enforcement
- Root account protection
- SSH hardening
- Firewall configuration
- Disabling unused services
- System updates and patch management
- File permission security
- Log monitoring

## Linux Hardening Checklist
- Root login disabled
- Least privilege applied
- Firewall configured
- Unused services disabled
- File permissions secured
- Logs monitored regularly

## Deliverables
- Linux Server Hardening Report (PDF)
